## 程序的表示

好吧，评估器消耗了什么？它消耗的是程序。因此，我们需要弄清楚如何表示程序。

当然，计算机一直在表示程序。当我们编写代码时，我们的文本编辑器保存着程序源代码。磁盘和内存中的每个可执行文件都是程序的一种表示。当我们访问网页时，它会发送一个 JavaScript 程序。这些都是计算机中表示的程序。但所有这些对我们来说都有些不方便，我们将在下一刻提出一个更好的表示方法。

在考虑表示之前，让我们先思考一下我们要表示什么。以下是一些示例（算术）程序：

```
1
0
-1
2.3
1 + 2
3 + 4
```

已经有一个问题。我们应该如何编写我们的程序？你可以看到这个趋势：我们应该将 1 和 2 的和写成

```
1 + 2
```

或者作为

```
(+ 1 2)
+ 1 2
1 2 +
```

等等。（至于基本数字的数制，我们甚至可以询问应该使用哪种：例如，我们应该写`3`还是`III`？如果你真的想这么做，你可以[使用后者编程](https://www.google.com/url?q=https://github.com/shriram/roman-numerals&sa=D&source=editors&ust=1695232021096337&usg=AOvVaw196-G7CiljdYG4eyIDGvwA)。）

因此，这些都是很好的考虑人类因素的问题。但就目前而言，这些因素在理解语言背后的模型方面是一个干扰。因此，我们需要一种方法来表示所有这些不同的程序，同时忽略这些区别。
