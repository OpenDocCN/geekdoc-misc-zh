# 评估局部绑定

大多数编程语言都有局部绑定的概念。这里有两个词，我们将把它们分开来讨论：

+   绑定意味着将名称与值关联起来。例如，当我们调用一个函数时，调用行为将形式参数与实际值关联（绑定）。

+   局部意味着它们限制在程序的某个区域，并且在该区域之外不可用。

例如，在许多语言中，我们可以写一些像这样的东西

```
fun f(x):
  y = 2
  x + y
```

这看起来足够清晰。但是这里有一个更微妙的程序：

```
fun f(x):
  for(y from 0 to 10):
    print(x + y)
  y
```

这是合法的吗？这取决于`y`是否仍然“活跃”或“有效”或“可见”或你喜欢的任何其他短语；正式地说，我们会说，这取决于`y`是否在作用域内。具体来说，我们会问最后一个`y`是否是`for`中发生的绑定的绑定实例。

这很复杂！许多语言做了一些相当奇怪、复杂，当然也是不直观的事情，正如你在《神秘语言》中看到的。这些奇怪的事情并不是 SMoL 的一部分；如果有什么的话，它们是对 SMoL 的违反。
