## 抽象语法

这引导我们来到 SImPl（标准实现计划）的第一部分：创建所谓的抽象语法。在抽象语法中，我们表示输入的本质，忽略表面的语法细节。因此，在抽象语法中，所有上述程序都将有完全相同的表示。

抽象语法是程序在计算机中的表示。我们可以使用许多种类的数据作为表示，所以让我们考虑我们可能想要表示的程序类型。为了简单起见，我们假设我们的语言只有数字和加法；一旦我们能够处理这一点，处理其他操作就会变得容易。以下是一些样本（表面语法）程序：

```
1
2.3
1 + 2
1 + 2 + 3
1 + 2 + 3 + 4
```

在传统的算术表示法中，当然，我们必须担心运算的顺序以及哪些运算比其他运算优先。在抽象语法中，这也是我们想要忽略的另一个细节；我们将假设我们在内部使用完全括号化的表达式，其中所有这些问题都已解决。因此，上面的最后两个表达式可以写成

`(1 + 2) + 3` 或 `1 + (2 + 3)`

```
1 + ((2 + 3) + 4)
```

那么，观察一下加法运算的每一侧都可以是一个完整的表达式。这给我们一个强烈的暗示，即我们应该使用什么样的内部表示：一棵树。确实，抽象语法树（AST）的使用如此普遍，以至于缩写 AST 通常被直接使用而不加解释；你可以预期在关于这个主题的书籍、论文、博客文章等中看到它。
